# Stage 3: jsreverser 深度逆向

使用此 playbook 进行第三阶分析：当前两阶手段无法解决问题时，启动 jsreverser-mcp 进行深度逆向。

## 何时进入第三阶

信号明确后才进入，不要过早升级：

- 代码经过 VMP 虚拟化保护，手动扣代码无法理解控制流
- 多层混淆（obfuscator.io + 自研 + 字符串加密），无法定位加密入口
- WASM 签名，JS 层只有胶水代码，核心逻辑在 .wasm 中
- 代码动态生成（eval 链），每次运行代码结构不同
- 环境检测极其严格，补环境输出始终与浏览器不一致且无法定位差距

## 核心原则

jsreverser 是核武器级的工具。用它不是为了炫技，是因为前两阶真搞不定。

## 工作方法

### 第一步：确认前两阶已穷尽

在启动 jsreverser 之前，确认：

- [ ] 第一阶：裸 Python 请求被拦截，且拦截不是 headers/cookies 能解决的
- [ ] 第二阶：Playwright 已定位到加密参数所在的 JS 文件/代码块
- [ ] 第二阶：尝试扣代码但代码被 VMP/重度混淆保护
- [ ] 第二阶：或尝试补环境但输出始终不一致
- [ ] 记录前两阶的具体失败原因和证据

### 第二步：准备逆向输入

将以下材料准备好：

1. **目标 JS 文件**：包含加密逻辑的 JS 文件（可以是从浏览器 Sources 中保存的）
2. **固定输入输出对**：使用相同输入在浏览器中获得的标准输出
3. **调用入口**：哪个函数是加密的入口？参数是什么？
4. **上下文信息**：这段代码所在的页面 URL、请求的 API 地址

### 第三步：启动 jsreverser-mcp

```python
# 使用 jsreverser-mcp 分析目标 JS
# jsreverser 擅长处理：
# - VMP 虚拟化还原（AST 分析 + 符号执行）
# - 混淆代码反混淆（控制流平坦化、字符串加密、常量折叠）
# - WASM 反编译和分析
# - 动态跟踪参数生成逻辑
```

根据 jsreverser 的分析结果：

1. 如果 jsreverser 还原出了可读的 JS 逻辑 → 作为 Node.js helper 接入 Python
2. 如果 jsreverser 还原出了纯算法逻辑 → 用 Python 重写
3. 如果 jsreverser 可以脱壳 WASM → 提取算法用 Python 实现或保留最小 WASM runtime

### 第四步：验证还原结果

无论 jsreverser 输出什么形式，验证标准不变：

- 固定输入 → 输出与浏览器一致
- 同一输入多次运行输出稳定
- 至少两条不同的输入都验证通过
- 能够集成到 Python 采集流程中

### 第五步：输出交付

根据还原程度选择交付方式：

- **最佳**：纯 Python 实现（无外部依赖）
- **次选**：Python + 最小 JS helper（不超过 200 行）
- **兜底**：Python + WASM runtime（保留必要的 .wasm 调用）

## 与第二阶的关系

第二阶和第三阶不是割裂的：

- 第二阶已经完成了加密点的定位和请求结构的分析
- 第三阶聚焦于"那段被定位到但扣不出来的代码"
- 第三阶的输出可以直接替换第二阶中扣代码失败的部分
- 第二阶的 Playwright 调试成果（请求结构、参数列表）在第三阶仍然有用


## 验证清单

在阶段 3 完成前逐项确认：

- [ ] 前两阶已穷尽，升级日志已写
- [ ] 目标 JS 文件已保存到本地
- [ ] 固定输入输出对已准备（至少 2 组）
- [ ] jsreverser-mcp 分析已完成
- [ ] 还原结果：固定输入 A 输出与浏览器一致
- [ ] 还原结果：固定输入 B 输出与浏览器一致
- [ ] 同一输入多次运行输出稳定
- [ ] 还原代码已集成到 Python 采集器
- [ ] 至少两条数据成功获取

如果 jsreverser 无法还原：
- [ ] 已向用户报告阻塞原因
- [ ] 已给出替代方案建议（降级、换工具、找人工分析）

## 常见陷阱

- 不要在第一阶没走完时就跳进 jsreverser
- 不要在还看不懂 Playwright 的 Network 面板时就寄希望于 jsreverser
- 不要对每个项目都用 jsreverser——它有学习成本
- 不要把 jsreverser 输出直接当最终答案，仍需固定输入验证
- 不要在 jsreverser 分析期间丢掉前两阶积累的请求结构认知
- 不要期望 jsreverser 能100%还原——能拿到参数生成逻辑就够了

## 退出条件

**成功退出：**
- 参数生成逻辑在本地可稳定运行
- 集成到 Python 采集流程后成功率100%（至少两条数据验证）
- 代码可读、可维护

**阻塞退出：**
- jsreverser 无法还原核心逻辑
- WASM 中使用了硬件级保护
- 需要真实的浏览器环境才能运行（这种情况极少）
- 此时应向用户报告阻塞原因和建议方案
