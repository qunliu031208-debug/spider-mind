# Workflow Overview

使用此文件作为从零到一的最短端到端地图。

## 启动门

动手前：

- 跑裸 Python 请求确认目标响应情况
- 根据响应结果做阶位决策（见 `strategy-triage.md`）
- 创建项目骨架（目录、venv、日志、配置）
- 声明本轮预计的最终交付形态

## 阶段 0：目标指纹

在写代码之前先给目标画像：

- 公开接口还是需要登录
- 反爬类型：IP 封禁、UA 检测、验证码、JS 挑战、加密参数、WASM、字体反爬
- 响应格式：JSON / HTML / XML / 二进制
- 分页方式：页码 / 游标 / 滚动加载
- 是否有频率限制
- 目标数据量预估

## 阶段 1：裸 Py 探路

执行（详见 `stage-1-pure-py.md`）：

- 构造最小请求，发 GET/POST
- 分析响应状态码和内容
- 逐个添加必要 headers（UA、Referer、Cookie 等）

退出条件：
- 数据到手 → 跳过逆向，直接进入阶段 4 交付
- 被拦截 → 记录拦截特征，进入阶段 2

## 阶段 2：补环境扣代码

执行（详见 `stage-2-env-patch.md`）：

1. Playwright 打开目标页，Network 面板观察真实请求
2. 定位加密参数在哪段 JS 中生成
3. 扣下加密逻辑代码
4. 本地 Node.js 补环境运行
5. 固定输入验证输出一致性
6. Python 通过 subprocess 调用 Node.js helper

**代码注释要求**：所有 Python 关键代码段必须添加详细的中文注释，解释每一步的意图和原理。

退出条件：
- Node.js helper 稳定输出 → 进入阶段 4 交付
- VMP / 重度混淆 / WASM 扣不出来 → 进入阶段 3

## 阶段 3：jsreverser 深度逆向

执行（详见 `stage-3-jsreverser.md`）：

1. 启动 jsreverser-mcp
2. 对混淆/VMP/WASM 做深度分析
3. 还原为纯 Python 或最小 JS helper
4. 验证可重复性

退出条件：
- 参数生成逻辑在本地稳定运行 → 进入阶段 4 交付

## 阶段 4：交付

执行（详见 `project-scaffold.md`）：

- 完整项目目录结构
- 虚拟环境 + requirements.txt
- 日志模块 + 配置文件
- 采集脚本（含 helper 调用，全部带中文注释）
- 数据存储（JSON/CSV/SQLite 按需）
- 至少两条数据验证通过

## 阶段 5：代码质量检查

执行（详见 `code-quality.md`）：

交付前必须完成的检查：

- 从零开始重建环境，确认项目可独立运行（`pip install` + `python main.py`）
- 检查代码中的编码问题（UTF-8 声明、文件读写编码、中文注释乱码）
- 检查异常处理和边界情况（网络超时、空数据、分页终止）
- 检查 hardcode 和敏感信息（路径、Cookie、Token 不应在代码中）
- 运行 `python main.py` 至少完整跑通两页数据
- Node.js helper（如有）独立自测通过

**禁止**：未经质量检查直接交付。

## 阶段 6：开发文档

执行（详见 `dev-doc.md`）：

项目交付必须附带一份开发文档 `DEV.md`，内容包括：

- 项目环境要求（Python 版本、Node.js 版本、系统依赖）
- 依赖库清单及版本（requirements.txt 已覆盖，文档中说明关键库用途）
- 从零到一的完整推理过程（每阶段做了什么、为什么这样做）
- 关键代码片段及说明（加密核心逻辑、补环境要点）
- 遇到的重要难点及解决方案
- 项目执行流程图（启动 → 采集 → 存储 的完整链路）
- 常见问题及排查方法

**目的**：让接手项目的开发者不看代码就能理解整个逆向过程，快速上手维护。

## 阶段间的切换规则

- 每阶先穷尽当前手段再升级，不要跳阶
- 升级和降级的详细规则见 `escalation-ladder.md`
- 升级时记录当前阶的失败原因
- 如果下一阶也失败，回到上一阶重新验证基线
- 不要在前一阶没走完的情况下跳进更重的工具

## 工具选用速查

| 工具 | 用途 | 适用阶段 |
|------|------|----------|
| Python requests | 裸请求探路 | 阶段 1 |
| Python httpx | 异步并发采集 | 阶段 1、交付 |
| Playwright | 浏览器调试、网络观察 | 阶段 2 |
| Node.js | 本地运行扣下来的 JS | 阶段 2 |
| jsreverser-mcp | 深度逆向混淆/VMP/WASM | 阶段 3 |
