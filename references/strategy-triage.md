# 策略分诊

使用此文件在新目标到手时快速决策进入哪一阶。

## 前置：结构化侦察

在做阶位决策之前，先完成结构化侦察（详见 `recon.md`），明确：
- 站点架构（SSR / SPA）
- 数据来源（API / HTML / WebSocket）
- 反爬手段（IP / JS Challenge / 加密参数 / 验证码 / WASM）
- 请求链路依赖

侦察结论是阶位决策的**输入**，不要跳过。

## 决策树

```
拿到目标 URL
   │
   ├→ 先做结构化侦察（recon.md）
   │
   ├→ 发一个裸 GET/POST
   │
   ├→ 200，返回正常数据 → 第一阶：裸 Py 探路
   │   └→ 成功获取数据 → 直接脚手架交付
   │
   ├→ 触发验证码 → 验证码分流
   │   ├→ 简单数字/字母验证码 → 验证码流水线（截图 + AI 预处理 + ddddocr）
   │   │   └→ 识别成功 → 继续采集流程
   │   │   └→ 识别失败（重试 3 次）→ 降级处理
   │   ├→ 滑块验证 → Playwright 模拟拖拽
   │   └→ 商业验证码（极验/顶象）→ 分析是否有 API 绕过
   │
   ├→ 403/406/412 等拦截 → 分析拦截特征
   │   ├→ 仅缺少基础 headers（UA、Referer）→ 第一阶补充 headers
   │   ├→ 需要 Cookie/Token，但可从页面或登录获取 → 第一阶 + session
   │   ├→ JS 生成签名参数 → 第二阶：补环境扣代码
   │   └→ Cloudflare / CDN 盾 → 评估是否值得突破
   │
   ├→ 200 但返回加密/混淆数据 → 分析数据格式
   │   ├→ 简单编码（base64、hex）→ 第一阶 + 解码
   │   ├→ JS 解码逻辑 → 第二阶扣解码代码
   │   └→ 字体反爬映射 → 第二阶 + 字体解析
   │
   └→ 请求成功但参数不知道怎么生成 → 进入 Playwright 网络观察
       ├→ 参数可在 Sources 中定位 → 第二阶扣代码
       └→ VMP/重度混淆/WASM → 第三阶 jsreverser
```

## 验证码分流策略

| 验证码类型 | 识别方案 | 预处理重点 | 成功率预期 |
|-----------|----------|------------|-----------|
| 4-6 位数字+字母 | ddddocr | 二值化即可 | 80-95% |
| 纯数字（4-6 位） | ddddocr | 放大 + 二值化 | 90-98% |
| 中文验证码 | ddddocr | 放大 4-5x + 锐化 | 60-80% |
| 滑块验证 | Playwright 拖拽 | 计算滑块距离 | 依赖实现质量 |
| 点选验证 | ddddocr (det=True) | 目标检测模式 | 50-70% |
| 极验/顶象/网易 | 不直接处理 | 优先找 API 绕过 | — |

**验证码决策规则**：
- ddddocr 能搞定的 → 流水线自动处理，不中断采集
- ddddocr 搞不定的 → 重试 3 次 → 记录日志 → 降级（人工/跳过）
- 商业验证码 → 不在此 skill 的主攻范围，优先找其他数据源

## 阶位升级信号

| 当前阶 | 升级信号 | 下一阶 |
|--------|----------|--------|
| 一阶 | 请求被拦截且无法通过 headers/cookies 解决 | 二阶 |
| 一阶 | 响应数据被加密且解码逻辑在 JS 中 | 二阶 |
| 一阶 | 触发滑块或点选验证，ddddocr 无法处理 | 二阶（Playwright） |
| 二阶 | 代码严重混淆，无法定位加密逻辑 | 三阶 |
| 二阶 | VMP 虚拟化保护，手动提取不可行 | 三阶 |
| 二阶 | WASM 签名，无法直接扣 JS | 三阶 |

## 阶位降级信号

| 当前阶 | 降级信号 | 操作 |
|--------|----------|------|
| 二阶 | 发现请求其实只需要正确的 headers | 回到一阶，直接 requests |
| 二阶 | 验证码可通过预处理 + ddddocr 解决 | 加验证码流水线，不回退阶位 |
| 三阶 | 发现加密逻辑在某个清晰的 JS 文件中 | 降到二阶，直接扣代码 |

## 每阶启动前的自检问题

侦察阶段（阶段 0）：
- [ ] 站点是 SSR 还是 SPA？数据从哪来？
- [ ] 有哪些反爬手段？最难的是哪个？
- [ ] 请求链路是怎样的？（Cookie → Header → Token）
- [ ] 是否有验证码？什么类型？

第一阶启动前：
- [ ] 目标 URL 可访问吗？
- [ ] 需要登录态吗？
- [ ] 响应是 JSON 还是 HTML？
- [ ] 如果触发验证码，类型是什么？能用 ddddocr 处理吗？

第二阶启动前：
- [ ] 第一阶在哪一步被拦住了？错误信息是什么？
- [ ] 被拦的究竟是哪个请求（用浏览器 Network 确认）？
- [ ] 那个请求的哪个参数是动态生成的？
- [ ] 中间是否有验证码环节？

第三阶启动前：
- [ ] 第二阶的 Playwright 调试定位到具体哪段 JS 了吗？
- [ ] 混淆类型是 obfuscator.io、JSVMP 还是自研方案？
- [ ] 是否涉及 WASM？如果能直接调用 .wasm 就不需要完全还原？
