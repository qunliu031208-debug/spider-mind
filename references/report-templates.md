# Report Templates

使用此文件规范每个阶段的输出格式和最终交付报告的结构。

## 阶段报告格式

每完成一个阶段，输出简短的阶段性报告，让过程可追溯：

### 阶段 1 报告

```markdown
## 阶段 1：裸 Py 探路报告

**目标**: {URL}
**请求方式**: GET/POST
**响应状态码**: {200/403/...}
**响应格式**: {JSON/HTML/...}
**关键 headers**: {列出起作用的 headers}
**结论**: {成功 / 需要进入二阶}

**验证码处理**（如触发）:
- 验证码类型: {数字/字母/...}
- 识别方案: Playwright 截图 + PIL 预处理 + ddddocr
- 识别结果: {通过 / 失败（重试 N 次）}

**如果失败**:
- 拦截类型: {403/加密/JS 挑战/...}
- 尝试过的 headers: {UA/Referer/Cookie/...}
- 失败原因推测: {...}
```

### 阶段 2 报告

```markdown
## 阶段 2：补环境扣代码报告

**Playwright 定位结果**:
- 目标 API: {URL}
- 加密参数: {sign/token/encrypt/...}
- 加密代码位置: {JS 文件名 + 函数名}
- 代码混淆程度: {无/轻度/中度/严重}

**补环境结果**:
- 扣下来的代码行数: {N}
- 补的环境对象: {window/navigator/document/...}
- 固定输入测试: {通过/失败}
- 与浏览器输出一致性: {一致/有差异}

**结论**: {成功交付 / 需要进入三阶}

**如果失败**:
- 失败原因: {VMP/混淆/WASM/环境差异/...}
```

## 最终交付报告

项目交付时输出以下报告：

```markdown
# {项目名称} 交付报告

## 目标信息
- 目标站点: {URL}
- 数据类型: {商品/新闻/用户/...}
- 数据量级: {预估总条数}
- 是否需要登录: {是/否}

## 技术方案
- 最终阶位: {阶段 1/2/3}
- 采集方式: {纯 Python / Python + Node.js helper / Python + WASM}
- 加密参数处理: {无 / Node.js helper / jsreverser 还原 / ...}
- 分页方式: {页码/游标/滚动}

## 项目结构
```
{项目名}/
├── main.py
├── config.py
├── collector/
│   ├── client.py       # HTTP 请求封装
│   ├── parser.py       # 数据解析
│   └── pipeline.py     # 数据存储
├── helper/             # {有/无 Node.js helper}
│   ├── env.js
│   ├── encrypt.js
│   └── bridge.js
└── storage/data/       # 数据输出
```

## 使用方法
```bash
cd {项目名}
python -m venv venv
venv\Scripts\pip install -r requirements.txt
copy .env.example .env    # 填入 Cookie
python main.py
```

## 阶段 6 开发文档交付

开发文档 `DEV.md` 需包含（详见 `dev-doc.md` 模板）：

- [ ] 项目概述和目标信息
- [ ] 环境要求和快速启动命令
- [ ] 逆向过程时间线
- [ ] 关键技术细节（加密分析、请求结构、响应解析）
- [ ] 遇到的重要难点及解决方案
- [ ] 项目结构说明
- [ ] 执行流程图
- [ ] 常见问题 FAQ

开发文档产出后，复制到项目根目录 `DEV.md`。
