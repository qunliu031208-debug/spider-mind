# 结构化侦察分析（Stage 0 升级版）

## 概述

在接受任何采集任务后，不急于发请求，先用 5-10 分钟完成结构化侦察。
侦察的目的是**建立对目标的系统理解**，而非立即拿到数据。

侦察结论将直接决定后续的阶位选择和工具组合。

## 侦察维度

### 1. 站点架构识别

在浏览器中打开目标网站，按 F12：

| 观察点 | 判断方法 | 结论 |
|--------|----------|------|
| 首屏渲染 | 查看页面源代码（Ctrl+U），搜索关键数据是否在 HTML 中 | SSR：数据在源码中 / SPA：源码是空壳 |
| 前端框架 | F12 → Elements，看 `<html>` 标签的 `data-*` 属性或 `<div id="app">` | Vue / React / 原生 |
| 数据加载 | Network → XHR/Fetch，刷新页面观察 API 请求 | API 路径、请求参数、响应格式 |
| 路由方式 | URL 是否带 `#`，页面切换是否刷新 | Hash 路由 / History 路由 |

**关键输出**：
- 数据来源：SSR 直接渲染 / API 异步加载 / WebSocket 推送
- 如果能找到 API 接口，记录接口 URL 和参数格式

### 2. 反爬指纹识别

| 反爬类型 | 识别方法 | 严重程度 |
|----------|----------|----------|
| IP 封禁 | 多次不带代理请求，观察何时被 ban | 低（换代理即可） |
| UA 检测 | 用 Python 默认 UA 测试，观察是否返回 403 | 极低（加 UA 即可） |
| Cookie 检测 | 不带 Cookie 请求，观察是否返回登录页或错误 | 低 |
| JS Challenge | 响应的 `<script>` 中含有 `__jsl_clearance` / `challenge` 等 | 中 |
| 加密参数 | 请求参数中有 `sign` / `token` / `_s` / `encrypt` 等动态值 | 中高 |
| 验证码 | 响应中包含验证码图片或验证码服务 URL | 中（有专项处理方案） |
| Cloudflare | 响应头包含 `cf-ray`，状态码 403/503 | 高 |
| 字体反爬 | 页面正常渲染但数据在源码中是乱码 | 中高 |
| WASM 加密 | Network 中有 `.wasm` 文件，请求参数由 WASM 生成 | 高 |
| 浏览器指纹 | JS 中检测 `navigator` / `webgl` / `canvas` 等 | 高（需要补环境或 jsreverser） |

**关键输出**：
- 主要反爬手段（可能多种共存）
- 预估处理难度和所需时间

### 3. 请求链路追踪

用浏览器 F12 → Network 面板，刷新页面，追踪完整请求链：

```
初始请求 (document)
    ↓ Set-Cookie: session_id=xxx
    ↓ 可能触发 JS Challenge
    ↓
JS Challenge 通过后 → Set-Cookie: __jsl_clearance=xxx
    ↓
XHR API 请求 → 携带 session + clearance + sign + timestamp
    ↓
返回 JSON 数据
```

**关键输出**：
- 请求链上每一步的依赖关系
- 哪些 Cookie/Header 是从上一步获取的
- 哪个参数是"最难生成"的

### 4. 数据格式分析

| 数据类型 | 判断方法 | 解析策略 |
|----------|----------|----------|
| JSON | `Content-Type: application/json` | `resp.json()` |
| HTML（SSR） | 返回完整 HTML，数据在标签中 | BeautifulSoup / Parsel |
| HTML（需要渲染） | 返回空壳或占位符 | 找 API 接口 |
| XML | `Content-Type: application/xml` | lxml / xml.etree |
| Base64 编码 | 返回乱码但可被 `base64.b64decode` 解码 | 先解码再处理 |
| 加密文本 | 返回乱码，解码失败 | 需要逆向解密逻辑 |

### 5. 分页与频率限制

| 观察点 | 方法 |
|--------|------|
| 分页方式 | URL 中有 `page=` / `offset=` / `cursor=`？还是滚动加载？ |
| 总数据量 | 响应中是否有 `total` / `totalCount` 字段？ |
| 单页数据量 | 每页返回多少条？ |
| 频率限制 | 快速连续发 5 个请求，观察是否触发限流 |

## 侦察报告模板

完成侦察后，输出结构化的侦察报告：

```markdown
# 侦察报告 — {target_name}

## 站点画像
- **URL**: {url}
- **架构**: {SSR / SPA (Vue/React)}
- **数据来源**: {SSR HTML / API 接口 / 需渲染}
- **API 接口**: {api_url}

## 反爬评估
- **主要手段**: {列出}
- **难度预估**: {低/中/高}
- **预计阶位**: {一阶/二阶/三阶}

## 请求链路
{简述从上到下的依赖关系}

## 数据规模
- **总数据量**: {预估值}
- **分页方式**: {页码/游标/滚动}
- **频率限制**: {有/无，限制多少}

## 推荐方案
- **建议阶位**: {N}
- **关键挑战**: {最难的一个点}
- **预计耗时**: {分钟数}
```
